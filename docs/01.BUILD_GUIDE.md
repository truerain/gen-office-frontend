# Gen-Office 프로젝트 실행 가이드 (레거시)

## ⚠️ 중요: 스타일이 적용되지 않는 문제 해결

MDI 페이지나 다른 페이지에서 스타일이 적용되지 않는다면, **theme 패키지가 빌드되지 않았기 때문**입니다.

## 📋 올바른 실행 순서

### 1. 의존성 설치
```bash
pnpm install
```

### 2. 모든 패키지 빌드 (필수!)
```bash
# 전체 패키지 빌드
pnpm build

# 또는 개별 패키지 빌드
pnpm build --filter=@gen-office/design-tokens
pnpm build --filter=@gen-office/theme
pnpm build --filter=@gen-office/utils
pnpm build --filter=@gen-office/primitives
pnpm build --filter=@gen-office/mdi
```

**중요**: `@gen-office/theme` 패키지를 빌드해야 CSS 변수들이 `dist/index.css`에 포함됩니다!

### 3. Demo 앱 실행
```bash
pnpm demo
```

## 🔍 스타일 문제 확인 방법

### 증상
- MDI 페이지가 스타일 없이 보임
- spacing, color 등이 적용되지 않음
- 브라우저 개발자 도구에서 CSS 변수가 정의되지 않음

### 확인 방법
```bash
# theme 패키지의 dist 폴더 확인
ls packages/theme/dist/

# 다음 파일들이 있어야 합니다:
# - index.mjs
# - index.css  <-- 이 파일이 중요!
# - index.d.ts
```

### 해결 방법
```bash
# theme 패키지만 다시 빌드
pnpm build --filter=@gen-office/theme

# 또는 전체 재빌드
rm -rf packages/*/dist
pnpm build
```

## 📦 패키지별 빌드 순서

의존성 순서대로 빌드해야 합니다:

1. **design-tokens** (다른 패키지에 의존성 없음)
2. **theme** (design-tokens에 의존)
3. **utils** (의존성 없음)
4. **primitives** (theme, utils에 의존)
5. **mdi** (primitives, theme에 의존)
6. **datagrid** (primitives, theme에 의존)

## 🎨 CSS 변수 확인

빌드가 정상적으로 되었다면, 브라우저 개발자 도구에서 다음 CSS 변수들이 정의되어 있어야 합니다:

```css
:root {
  --spacing-xs: 0.25rem;
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --spacing-lg: 1.5rem;
  --spacing-xl: 2rem;
  
  --color-primary: #a50034;
  --color-background: #ffffff;
  --color-surface: #f8f9fa;
  --color-text: #212529;
  --color-border: #dee2e6;
  
  --font-size-sm: 0.875rem;
  --font-size-base: 1rem;
  --font-size-lg: 1.125rem;
  
  --border-radius-sm: 0.25rem;
  --border-radius-md: 0.5rem;
  
  /* ... 등등 */
}
```

## 🚀 빠른 시작 (한 번에)

```bash
# 1. 설치 및 빌드
pnpm install && pnpm build

# 2. Demo 실행
pnpm demo

# 브라우저에서 http://localhost:5173 접속
# 상단 메뉴에서 "MDI" 클릭
```

## 🐛 일반적인 문제 해결

### 문제 1: "Cannot find module '@gen-office/theme'"
```bash
# 해결: theme 패키지 빌드
pnpm build --filter=@gen-office/theme
```

### 문제 2: CSS 변수가 undefined
```bash
# 해결: theme 빌드 확인
ls packages/theme/dist/index.css

# 파일이 없으면 빌드
pnpm build --filter=@gen-office/theme
```

### 문제 3: 전체 재빌드 필요
```bash
# 모든 dist 폴더 삭제 후 재빌드
pnpm clean
pnpm install
pnpm build
```

## 📝 개발 워크플로우

### 패키지 수정 시
```bash
# 1. 패키지 수정 (예: primitives)
# 2. 해당 패키지 재빌드
pnpm build --filter=@gen-office/primitives

# 3. Demo 앱 재시작 (Vite가 자동으로 리로드)
```

### 새 컴포넌트 추가 시
```bash
# 1. 컴포넌트 작성
# 2. 패키지 빌드
pnpm build --filter=@gen-office/primitives

# 3. Demo 앱에서 사용
import { NewComponent } from '@gen-office/primitives';
```
