# GenOffice Frontend 빌드 가이드

이 문서는 GenOffice Frontend 리포지토리의 설치, 빌드, 실행, 트러블슈팅 절차를 정리합니다.

AI 코딩 작업 규칙은 루트의 `AGENTS.md`를 따르고, 이 문서는 빌드와 실행 문제 해결에만 집중합니다.

## 요구 사항

- Node.js 20 이상
- pnpm 9 이상

현재 루트 `package.json` 기준:

```json
{
  "packageManager": "pnpm@9.15.0",
  "engines": {
    "node": ">=20.0.0",
    "pnpm": ">=9.0.0"
  }
}
```

## 설치

리포지토리 루트에서 실행합니다.

```bash
pnpm install
```

## 빠른 시작

데모 앱 실행:

```bash
pnpm dev
```

Mock 모드 데모 앱 실행:

```bash
pnpm demo:mock
```

Storybook 실행:

```bash
pnpm storybook
```

## 전체 빌드

전체 워크스페이스 빌드:

```bash
pnpm build
```

이 명령은 Turbo를 통해 각 앱과 패키지의 `build` 스크립트를 실행합니다.

## 주요 검증 명령

```bash
pnpm build
pnpm lint
pnpm test
pnpm check:encoding
```

특정 앱 또는 패키지만 확인할 때는 `pnpm -C`를 사용합니다.

```bash
pnpm -C apps/demo build
pnpm -C packages/theme build
pnpm -C packages/ui build
pnpm -C packages/gen-grid exec tsc -p tsconfig.json --noEmit
pnpm -C packages/gen-grid-crud exec tsc -p tsconfig.json --noEmit
```

## 워크스페이스 구조

현재 워크스페이스 범위:

```yaml
packages:
  - apps/*
  - packages/*
```

앱:

- `apps/demo`: 통합 데모 앱
- `apps/storybook-all`: 통합 Storybook 앱

패키지:

- `packages/theme`: 디자인 토큰, 글로벌 스타일, 폰트
- `packages/ui`: 공통 UI 컴포넌트
- `packages/utils`: 공통 유틸리티
- `packages/mdi`: MDI 레이아웃
- `packages/gen-grid`: TanStack Table 기반 그리드
- `packages/gen-grid-crud`: 그리드 기반 CRUD 기능
- `packages/gen-grid-chart`: 그리드와 차트 연동
- `packages/gen-chart`: 차트 컴포넌트
- `packages/gen-datagrid`: 데이터 그리드 실험 또는 대체 구현
- `packages/tsconfig`: 공유 TypeScript 설정

## 빌드 순서

일반적으로는 `pnpm build`를 사용하면 됩니다. 수동으로 특정 패키지를 확인할 때는 의존성 흐름을 기준으로 아래 순서를 우선합니다.

```text
theme / utils / tsconfig
-> ui
-> mdi / gen-chart / gen-grid
-> gen-grid-crud / gen-grid-chart
-> apps/demo / apps/storybook-all
```

## 스타일 문제 확인

화면에서 색상, spacing, font, component style이 적용되지 않으면 먼저 `theme`과 사용하는 패키지의 `dist` 산출물을 확인합니다.

```bash
pnpm -C packages/theme build
pnpm -C packages/ui build
```

확인할 파일:

```text
packages/theme/dist/index.css
packages/ui/dist/index.css
```

데모 앱에서 패키지 CSS import가 누락되지 않았는지도 확인합니다.

## 일반적인 문제 해결

### Cannot find module '@gen-office/*'

워크스페이스 의존성이 설치되지 않았거나 대상 패키지 빌드 산출물이 없을 수 있습니다.

```bash
pnpm install
pnpm build
```

특정 패키지만 다시 빌드:

```bash
pnpm -C packages/theme build
pnpm -C packages/ui build
```

### 스타일이 적용되지 않음

`theme` 또는 `ui`의 CSS 산출물이 없는지 확인합니다.

```bash
pnpm -C packages/theme build
pnpm -C packages/ui build
```

그리고 앱 또는 패키지에서 필요한 CSS를 import하고 있는지 확인합니다.

### TypeScript 타입 오류가 특정 패키지에서만 발생

해당 패키지만 먼저 타입 검사를 실행합니다.

```bash
pnpm -C packages/gen-grid exec tsc -p tsconfig.json --noEmit
pnpm -C packages/gen-grid-crud exec tsc -p tsconfig.json --noEmit
```

### 인코딩 검사 실패

한국어 문서나 소스 파일을 수정한 뒤 인코딩 오류가 나면 다음 명령으로 확인합니다.

```bash
pnpm check:encoding
```

커밋 전 staged 파일만 검사하려면:

```bash
pnpm check:encoding:staged
```

UTF-8 BOM, replacement character, 한글 mojibake가 있으면 해당 파일을 UTF-8 without BOM으로 복구합니다.
