# 공통코드 관리 페이지 설계

## 1. 목적
- 운영자가 공통코드 그룹(`lkupClssCd`)과 상세코드(`lkupCd`)를 조회/등록/수정/비활성화하는 관리자 화면이다.
- 백엔드 API 문서 `backend/docs/api/admin/공통코드관리.md` 스펙을 기준으로 마스터-디테일 구조를 제공한다.

## 2. 메뉴/라우팅 구조 (DATA_DRIVEN_MENU 기준)
- 메뉴 데이터 소스: `/api/app-menus`
- 페이지 연결 우선순위:
  - 1순위: DB `execComponent`
  - 2순위: 프론트 폴백 매핑(`menuData.ts > componentNameByMenuId`)
- 공통코드 페이지 컴포넌트 이름(권장): `CommonCodeManagementPage`
- 동적 로딩 등록 파일: `apps/demo/src/app/config/componentRegistry.dynamic.ts`

## 3. 운영 반영 시 필수 설정
### 3.1 DB 메뉴 설정
- 테이블: `app_menus`(운영 시스템 기준)
- 권장 값:
  - `menu_name`: Common Code Management
  - `parent_menu_id`: System 메뉴 ID
  - `exec_component`: `CommonCodeManagementPage`
  - `display_yn`: `Y`
  - `use_yn`: `Y`
  - `sort_order`: 운영 정책에 맞게 설정

### 3.2 프론트 폴백 매핑
- 파일: `apps/demo/src/app/menu/menuData.ts`
- 목적: DB 응답에 `execComponent`가 누락되거나 빈 값일 때 메뉴 진입 보장
- 권장 매핑:
  - `900600 -> CommonCodeManagementPage`
  - 아이콘: `ListTree` 또는 `TableProperties`

## 4. 권한/접근 정책
- 관리자 권한(예: `ROLE_ADMIN`)만 접근 가능해야 한다.
- 운영 점검 시 확인 항목:
  - 비관리자 계정 접근 차단(401/403)
  - 세션 만료 시 표준 에러 응답 반환

## 5. 화면 기능 설계 요약
- 경로(예정): `apps/demo/src/pages/admin/common-code/CommonCodeManagementPage.tsx`
- UI 구조:
  - 좌측: 코드 그룹(마스터) 목록/검색/CRUD
  - 우측: 선택된 그룹의 상세코드(디테일) 목록/검색/CRUD
- 그룹 검색 조건:
  - `lkupClssCd`, `lkupClssName`, `useYn`, `q`
- 상세 검색 조건:
  - `lkupCd`, `lkupName`, `useYn`, `q`
- 저장 정책:
  - 물리 삭제 미지원
  - 삭제 동작은 `useYn = N` 업데이트로 처리

## 6. API 연동 포인트
- 그룹 목록: `GET /api/mis/admin/lookups/classes`
- 그룹 단건: `GET /api/mis/admin/lookups/classes/{lkupClssCd}`
- 그룹 생성: `POST /api/mis/admin/lookups/classes`
- 그룹 수정: `PUT /api/mis/admin/lookups/classes/{lkupClssCd}`
- 상세 목록: `GET /api/mis/admin/lookups/{lkupClssCd}/items`
- 상세 단건: `GET /api/mis/admin/lookups/{lkupClssCd}/items/{lkupCd}`
- 상세 생성: `POST /api/mis/admin/lookups/{lkupClssCd}/items`
- 상세 수정: `PUT /api/mis/admin/lookups/{lkupClssCd}/items/{lkupCd}`

## 7. 입력 검증/에러 처리
- 필수값:
  - 그룹: `lkupClssCd`, `lkupClssName`, `useYn`
  - 상세: `lkupCd`, `lkupName`, `useYn`
- 규칙:
  - `lkupClssCd`, `lkupCd` 공백 금지
  - `useYn`은 `Y` 또는 `N`
  - `sortOrder`는 0 이상 정수 권장
- 저장 실패 처리:
  - `409 Conflict`: 중복 코드 안내 메시지 노출
  - 기타 오류: 공통 알림 + 서버 메시지(`code`, `messageKey`, `message`) 우선 노출

## 8. 장애 점검 체크리스트
1. 메뉴는 보이는데 페이지가 안 열림
- `/api/app-menus` 응답의 `execComponent` 확인
- `componentRegistry.dynamic.ts`에 동일 컴포넌트명 등록 여부 확인
- `menuData.ts` 폴백 매핑 확인

2. 페이지는 열리는데 목록이 비정상
- 그룹 API(`/lookups/classes`)와 상세 API(`/lookups/{lkupClssCd}/items`) 상태코드/응답 확인
- 선택된 그룹 키(`lkupClssCd`) 전달 누락 여부 확인
- 권한 오류(401/403) 여부 확인

3. 저장 시 실패
- 그룹 키/상세 키 중복으로 인한 `409`인지 확인
- `useYn`, `sortOrder` 입력 형식 확인
- 서버 에러 포맷(`code`, `messageKey`, `message`) 확인

## 9. 배포 후 검증 시나리오
1. 관리자 계정으로 System > Common Code Management 진입
2. 그룹 1건 생성 후 목록/단건 조회 확인
3. 생성한 그룹 하위 상세코드 1건 생성 후 즉시 조회 확인
4. 동일 키 재생성 시 `409` 충돌 메시지 확인
5. 그룹/상세 각각 수정 후 반영 확인
6. 삭제 액션 수행 시 `useYn = N` 비활성화 반영 확인
